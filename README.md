# Spring Boot vs SAP CAP – Side-by-Side Comparison

This repository is a training resource for experienced **Spring Boot / Spring MVC / Hibernate** developers learning **SAP CAP (Cloud Application Programming model)**. Both projects implement exactly the same REST API so you can compare how each concept translates between the two stacks.

---

## Projects

### 1. [spring-boot-mvc-hibernate](./spring-boot-mvc-hibernate/README.md)

Standard Spring Boot 3 application using Spring MVC, Spring Data JPA (Hibernate), Spring Security, and an H2 in-memory database. Exposes a plain REST/JSON API.

**Build & Run:**
```bash
cd spring-boot-mvc-hibernate
mvn spring-boot:run
# API at http://localhost:8080
```

### 2. [sap-cap-java](./sap-cap-java/README.md)

SAP CAP Java application using CDS for domain modelling and OData v4 as the transport protocol. The framework auto-generates CRUD, handles authentication, and enforces role restrictions from declarative annotations.

**Build & Run:**
```bash
cd sap-cap-java
mvn clean package          # first run downloads Node.js + @sap/cds-dk (takes a few minutes)
mvn -pl srv -am spring-boot:run
# OData service at http://localhost:8080/odata/v4/UserDepartmentService
```

## Side-by-Side Comparison

| Concept | Spring Boot                                                                                                                                                                                                                             | SAP CAP Java                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|---|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Authentication** | `InMemoryUserDetailsManager` with `HttpBasic` configured in [`SecurityConfiguration.java`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/config/SecurityConfiguration.java)                                          | `cds.security.mock.users` in [`application.yaml`](./sap-cap-java/srv/src/main/resources/application.yaml) – no Java config class needed                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Domain model** | `@Entity` + `@Embeddable` in [`entity/`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/entity/)                                                                                                                      | CDS `entity` + `type` in [`db/data-model.cds`](./sap-cap-java/db/data-model.cds); POJOs are generated at build time into `cds.gen.*`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Embedded type (Address)** | `@Embeddable` [`Address`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/entity/Address.java) + `@Embedded` on [`User.address`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/entity/User.java) – stored as flat columns | CDS `type Address` in [`data-model.cds`](./sap-cap-java/db/data-model.cds) – same flat-column storage, nested JSON in OData |
| **Enums** | Java [`EuCountry`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/entity/EuCountry.java) enum in `entity/`; stored as `VARCHAR` via `@Enumerated(EnumType.STRING)` on [`Address.country`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/entity/Address.java); request DTOs accept the enum name as a plain `String` | `type EuCountry : String enum { AUSTRIA; BELGIUM; … }` in [`data-model.cds`](./sap-cap-java/db/data-model.cds); CAP automatically applies `@assert.range` to enum-typed fields and rejects unknown values – no Java code needed |
| **Many-to-many relationship** | `@ManyToMany` + `@JoinTable(name="user_departments")` on [`User.departments`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/entity/User.java)                                                                        | Explicit [`UserDepartments`](./sap-cap-java/db/data-model.cds) entity with two `Association to` keys – no hidden join table                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **CRUD operations** | `JpaRepository` + `@RestController` in [`controller/`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/controller/)                                                                                                    | Built-in OData v4 CRUD from `entity … as projection on` in [`user-department-service.cds`](./sap-cap-java/srv/user-department-service.cds) – zero controller code needed                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Validation (service-level, simple)** | `@NotBlank`, `@Email` on DTO fields in [`dto/`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/dto/) + `@Valid` in [`UserController.java`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/controller/UserController.java) parameters | `@mandatory` (non-null/non-blank) and `@assert.format` (regex) declared directly on fields in [`data-model.cds`](./sap-cap-java/db/data-model.cds); enforced automatically by the CAP runtime – no Java handler needed |
| **Validation (persistence-level)** | `@NotBlank`, `@Email` on [`User`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/entity/User.java) entity fields – Hibernate Validator enforces these before every `INSERT`/`UPDATE` and generates `NOT NULL` DDL; `@Column(unique=true)` on `email` generates a DB unique constraint; uniqueness is also checked in [`UserService`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/service/UserService.java) via [`UserRepository.existsByEmail`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/repository/UserRepository.java) to return a clean HTTP 409 | `not null` keyword on entity fields generates a DB `NOT NULL` column; `@assert.unique: { email: [email] }` on the [`Users`](./sap-cap-java/db/data-model.cds) entity enforces uniqueness at the service layer before hitting the DB – no Java code needed |
| **Validation (complex / domain logic)** | `@EuCountry` custom Jakarta constraint on [`CreateUserRequest.country`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/dto/CreateUserRequest.java) / [`UpdateUserRequest.country`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/dto/UpdateUserRequest.java); [`EuCountryValidator`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/validation/EuCountryValidator.java) delegates to `EuCountry.valueOf()` | `@Before` handler [`UserServiceHandler.validateEuCountry()`](./sap-cap-java/srv/src/main/java/com/example/capdemo/handlers/UserServiceHandler.java) checks the country against known enum names. **Note:** because `address.country` is typed as the `EuCountry` CDS enum, CAP already rejects unknown values via `@assert.range` before this handler runs – the handler is kept to demonstrate the `@Before` pattern and could be removed in a real project |
| **Read-only custom operation (OData Function)** | `GET /api/users/count` (`@GetMapping`) in [`UserController.java`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/controller/UserController.java)                                                                      | `function getUsersCount() returns Integer` in [`.cds`](./sap-cap-java/srv/user-department-service.cds) + `@On(event=GetUsersCountContext.CDS_NAME)` in [`UserServiceHandler.java`](./sap-cap-java/srv/src/main/java/com/example/capdemo/handlers/UserServiceHandler.java). Called via `GET .../getUsersCount()`. <br/>**NOTE:** for a plain count, `getUsersCount()` may be redundant – OData provides `$count` natively: `GET /odata/v4/UserDepartmentService/Users/$count` returns the total number of users without any custom code. A custom Function is only needed when extra logic (filtering, aggregation) is required. |
| **Mutating custom operation (OData Action)** | `POST /api/users/{id}/assign-department` (`@PostMapping`) in [`UserController.java`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/controller/UserController.java)                                                   | `action assignUserToDepartment(userID, departmentID)` in [`.cds`](./sap-cap-java/srv/user-department-service.cds) + `@On(event=AssignUserToDepartmentContext.CDS_NAME)` in [`UserServiceHandler.java`](./sap-cap-java/srv/src/main/java/com/example/capdemo/handlers/UserServiceHandler.java). Called via `POST .../assignUserToDepartment`                                                                                                                                                                                                                                                                                |
| **Role-based access control** | `@PreAuthorize("hasRole('ADMIN')")` on methods in [`UserController`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/controller/UserController.java) and [`DepartmentController`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/controller/DepartmentController.java) | `@restrict: [{ grant: '*', to: 'admin' }, { grant: 'READ', to: 'user' }]` on entities in [`user-department-service.cds`](./sap-cap-java/srv/user-department-service.cds) |
| **Pagination** | `Pageable` query params (`page`, `size`, `sort`) resolved by Spring MVC's [`UserController`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/controller/UserController.java) via `@PageableDefault`; [`UserService.findAll(Pageable)`](./spring-boot-mvc-hibernate/src/main/java/com/example/springdemo/service/UserService.java) delegates to `JpaRepository.findAll(Pageable)` issuing a `LIMIT/OFFSET` query; response is a `Page<T>` envelope with `totalElements`/`totalPages` | OData built-in `$top`, `$skip`, `$orderby` query options on any entity in [`user-department-service.cds`](./sap-cap-java/srv/user-department-service.cds); no Java code needed. Example: `GET /odata/v4/UserDepartmentService/Users?$top=5&$skip=0&$orderby=lastName` |
| **HTTP protocol** | Plain REST/JSON                                                                                                                                                                                                                         | OData v4 at `/odata/v4/UserDepartmentService`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Build layout** | Single-module Maven                                                                                                                                                                                                                     | Multi-module Maven (root `pom.xml` + `srv/pom.xml`)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
